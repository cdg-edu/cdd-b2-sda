2. Elementos curriculares

Competencias clave

  • Competencia en comunicación lingüística
    • Comprende y elabora documentación técnica de forma clara, precisa y estructurada.
    • Interpreta enunciados, instrucciones y documentación técnica en distintos formatos (digital o textual).
  • Competencia digital
    • Utiliza herramientas digitales para el tratamiento, análisis y visualización de datos.
    • Gestiona fuentes abiertas de datos para obtener información relevante sobre el entorno.
  • Competencia matemática y competencias básicas en ciencia, tecnología e ingeniería
    • Aplica estructuras de datos y principios de lógica para resolver problemas de programación.
  • Competencia personal, social y de aprender a aprender
    • Participa activamente en el desarrollo del proyecto, asumiendo responsabilidades y gestionando su aprendizaje.
       

Competencias específicas

  • C1: Desarrollar aplicaciones web utilizando tecnologías y entornos adecuados
  • C2: Comprender y aplicar los principios de la programación estructurada y orientada a objetos
  • C3: Desarrollar autonomía, responsabilidad y colaboración en proyectos
  • C4: Resolver problemas de forma lógica y estructurada

Saberes Básicos

  • Fundamentos del paradigma orientado a objetos: clases, objetos, atributos, métodos, encapsulamiento.
  • Estructuras de control: condicionales, bucles.
  • Colecciones y estructuras de datos: arrays, listas, mapas.
  • Gestión de ficheros y formatos: lectura y procesamiento de CSV y JSON.
  • Uso de APIs y datos abiertos: conceptos de OpenData, análisis y visualización de datos.
  • Buenas prácticas de codificación: comentarios, nombres significativos, modularidad.
  • Documentación del proyecto: README, diagramas UML básicos, informes técnicos.
  • Pensamiento computacional y resolución de problemas reales.

Criterios de evaluación relacionados con las competencias específicas 

C1: Desarrollar aplicaciones web utilizando tecnologías y entornos adecuados

  • 1.1. Utiliza correctamente un entorno de desarrollo para programar una aplicación Java relacionada con datos medioambientales.
  • 1.2. Integra datos abiertos reales (formato CSV o JSON) en la aplicación mediante código funcional y adaptado.
  • 1.3. Diseña una interfaz de entrada/salida en consola clara y funcional para mostrar los datos analizados.
  • 1.4. Aplica buenas prácticas de programación (convenciones, nomenclatura, documentación básica) durante el desarrollo del proyecto.

C2: Comprender y aplicar los principios de la programación estructurada y orientada a objetos

  • 2.1. Define e implementa correctamente clases, atributos y métodos que representen conceptos del dominio (ej. mediciones, estaciones, sensores).
  • 2.2. Emplea estructuras de control de flujo y estructuras de datos adecuadas (arrays, listas, mapas) para resolver la lógica del programa.
  • 2.3. Organiza el código en módulos reutilizables y comprensibles, aplicando el encapsulamiento y la responsabilidad única.
  • 2.4. Interpreta y aplica principios de diseño como modularidad, legibilidad y mantenimiento del código.

C3: Desarrollar autonomía, responsabilidad y colaboración en proyectos

  • 3.1. Gestiona su propio trabajo de forma autónoma, cumpliendo con las fases del proyecto dentro del tiempo previsto.
  • 3.2. Participa activamente en el desarrollo, revisión y mejora del proyecto, mostrando iniciativa personal.
  • 3.3. Coopera con sus compañeros en tareas de revisión de código, validación de datos o toma de decisiones técnicas.

C4: Resolver problemas de forma lógica y estructurada

  • 4.1. Analiza una necesidad real (monitorización de calidad ambiental) y la traduce en una solución técnica organizada.
  • 4.2. Diseña y ejecuta un algoritmo que procesa y resume datos extraídos de fuentes abiertas.
  • 4.3. Evalúa y depura errores de ejecución y lógica durante el desarrollo del programa, proponiendo soluciones efectivas.
  • 4.4. Explica el funcionamiento de su solución, justificando las decisiones tomadas durante la implementación.
