Título. Descripción de la Sda

Programamos por el Planeta: Monitorización de la calidad del aire y del agua con datos abiertos

  • Etapa educativa y nivel al que se dirige: Módulo de Programación del primer curso del CFGS Desarrollo de Aplicaciones Web
  • Áreas, materias y /o módulos implicados:

    • Entornos de desarrollo: gestión de proyectos y control de versiones

    • Bases de datos: para la futura integración del almacenamiento de datos

    • Lenguajes de marcas: estructuración y presentación de los datos

  • Agrupamientos y organización espacial:

    • Grupos heterogéneos

    • Trabajo individual parcial

    • Gran grupo para retroalimentación común

  • Temporalización: 8 sesiones de 2 horas
  • Productos a desarrollar en las tareas y herramientas implicadas:

    • TAREA 1: Exploración de APIs abiertas de la Unión Europea (1 sesión - 2 horas)

      El alumnado investigará las distintas fuentes de datos abiertas que ofrece la Unión Europea relacionadas con el medioambiente. Se centrarán en encontrar APIs que proporcionen datos de calidad del aire o del agua con formatos accesibles (JSON, XML). 

      Actividades del alumnado:

      Buscar APIs relacionadas con el medio ambiente, aire o agua.
      Analizar los formatos disponibles y las rutas de acceso a los datos.
      Seleccionar una API y anotar parámetros, estructuras de respuesta, y ejemplos de endpoints.
      Compartir sus hallazgos mediante una infografía.

      Recursos

      European Data Portal, documentación oficial de APIs (por ejemplo: EEA – European Environment Agency), ordenadores con acceso a internet, cuaderno de trabajo digital (Google Docs o PDF editable).

    • TAREA 2: Diseño del flujo del programa y modelo de datos (1 sesión - 2 horas)

      Los estudiantes definirán qué tipo de datos van a obtener y cómo los organizarán. Representarán el flujo de su programa en pseudocódigo o mediante un diagrama de flujo y diseñarán las estructuras necesarias (arrays, objetos...).

      Actividades del alumnado:

      Identificar qué datos relevantes devolverá la API (e.g., calidad del aire, niveles de partículas, fecha, ubicación).
      Crear diagramas de flujo del funcionamiento del programa.
      Especificar las estructuras necesarias (objetos en JSON o diccionarios).
      Presentar su diseño al docente o a otros grupos para validación.
      Crear un vídeo tutorial de uso de alguna herramienta colaborativa como Trello.

      Recursos

      Draw.io o Lucidchart, pizarra colaborativa (Jamboard, Miro, Trello), IDE para programación en Java, esquema de trabajo (plantilla para estructuras de datos)

    • TAREA 3: Implementación de parseo de datos y modularización del código (2 sesiones - 4 horas)

      El código se organizará mediante funciones o clases. El alumnado trabajará en el tratamiento de los datos obtenidos para presentarlos de manera clara y legible, incluyendo filtrado por ubicación o tipo de contaminante.

      Actividades del alumnado:

      Dividir el programa en funciones: conexión, parseo, visualización.
      Aplicar filtros por ciudad o tipo de contaminante.
      Añadir validaciones: que el JSON tenga campos esperados.
      Comprobar el correcto funcionamiento del código mediante pruebas básicas.

      Recursos

      IDE, guía de buenas prácticas de programación estructurada, tests unitarios básicos.

    • TAREA 4: Visualización simple de resultados (texto o HTML) (2 sesiones - 4 horas)
      Se crea una interfaz simple (texto formateado o salida en archivo HTML) que permita mostrar los datos de forma más visual y comprensible, destacando valores importantes o anómalos.

      Actividades del alumnado:

      Crear una tabla en consola o HTML con los datos seleccionados.
      Resaltar valores críticos con colores o mensajes.
      Comprobar que la visualización sea legible y útil para cualquier usuario.
      Generar un archivo HTML local que pueda abrirse en navegador.

      Recursos

      Librerías para generación de HTML (si se elige esta vía), hoja de estilos básica (CSS), plantilla de tabla para mostrar resultados.

    • TAREA 5. Documentación y presentación del proyecto final (2 sesiones - 4 horas)
      El alumnado recopilará la información sobre el desarrollo del proyecto: objetivos, diseño, código fuente comentado, resultados obtenidos y dificultades encontradas. Presentarán el proyecto a sus compañeros/as y al profesorado. 

      Actividades del alumnado:

      Documentar el funcionamiento y estructura del programa.
      Redactar una memoria del proyecto con imágenes, código y resultados.
      Preparar una presentación oral del proyecto (5-10 min por grupo).
      Subir los materiales a la plataforma común del centro.

      Recursos

      Plantilla de memoria del proyecto (Word o Office 365), herramienta de presentación (Canva, Slides, PowerPoint), GitHub para subir los archivos del proyecto.
